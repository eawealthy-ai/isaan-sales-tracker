วิธีตั้งค่า Firebaseฐานข้อมูลกลางสำหรับระบบติดตามยอดขาย · โปรอีสาน ซีรีย์

Firebase คือ “ที่เก็บข้อมูลกลาง” ฟรีของ Google เมื่อตั้งค่าเสร็จ ทุกสาขาจะกรอกยอดขายจากมือถือ/คอมเครื่องไหนก็ได้ แล้ว ทีมผู้บริหารเห็นยอดรวมทุกสาขาแบบเรียลไทม์ · ใช้แพ็กเกจฟรี (Spark) เพียงพอมาก (ยอดขาย ~300 จาน/วัน ใช้พื้นที่นิดเดียว) · ใช้เวลาประมาณ 10 นาที

1 สร้างโปรเจกต์ ~2 นาที

  1. เปิด console.firebase.google.com แล้วเข้าสู่ระบบด้วยบัญชี Google (eawealthy@gmail.com ใช้ได้)
  2. กด Add project (สร้างโปรเจกต์)
  3. ตั้งชื่อ isaan-tracker → Continue
  4. ปิด Google Analytics (ไม่ต้องใช้) → Create project → รอสักครู่ → Continue

2 สร้างฐานข้อมูล (“ที่เก็บข้อมูล”) ~2 นาที

  1. เมนูซ้าย: Build → Realtime Database
  2. กด Create Database
  3. เลือกที่ตั้ง: Singapore (asia-southeast1) — ใกล้ไทยที่สุด
  4. เลือก Start in test mode → Enable
✅ ตอนนี้คุณมีฐานข้อมูลแล้ว จะเห็นลิงก์ด้านบนหน้าจอ เช่น
https://isaan-tracker-default-rtdb.asia-southeast1.firebasedatabase.app
นั่นคือ “ที่อยู่ฐานข้อมูล” (databaseURL) — จำไว้ใช้ในขั้นตอนที่ 4

3 ขอรหัสเชื่อมต่อ (config) ~2 นาที

  1. กด ⚙️ เฟือง มุมซ้ายบน → Project settings
  2. เลื่อนลงหา Your apps → กดไอคอน </> (Web)
  3. ตั้งชื่อเล่น isaan-web → Register app
  4. จะเห็นโค้ดกล่องหนึ่งขึ้นต้นด้วย const firebaseConfig = { ... } — เปิดค้างไว้ จะใช้ขั้นต่อไป

4 วางรหัสลงในไฟล์ ~2 นาที

เปิดไฟล์ index.html ด้วยโปรแกรมแก้ข้อความใดก็ได้ (เช่น TextEdit / Notepad / VS Code) หาส่วนนี้ใกล้ ๆ ด้านบน แล้ววางค่าจากขั้นตอนที่ 3:

  // วางค่าของคุณตรงนี้
  firebase: {
    apiKey:      "AIzaSy........",
    authDomain:  "isaan-tracker.firebaseapp.com",
    databaseURL: "https://isaan-tracker-default-rtdb.asia-southeast1.firebasedatabase.app",
    projectId:   "isaan-tracker",
    appId:       "1:....:web:...."
  }
⚠️ ค่า databaseURL สำคัญที่สุด — คัดลอกจากด้านบนหน้า Realtime Database (ขั้นตอนที่ 2) ถ้าในกล่อง config ไม่มี databaseURL ให้ก๊อปจากหน้าฐานข้อมูลมาใส่เอง

บันทึกไฟล์ แล้วเปิดใหม่ — แถบด้านบนต้องเปลี่ยนเป็น 🟢 เชื่อมต่อระบบซิงก์ออนไลน์

5 ตั้งกฎความปลอดภัย (ให้ใช้ได้เกิน 30 วัน) ~1 นาที

โหมด test mode จะล็อกอัตโนมัติหลัง ~30 วัน · ไปที่ Realtime Database → Rules ลบของเดิม วางอันนี้แทน แล้วกด Publish:

{
  "rules": {
    "sales": { ".read": true, ".write": true },
    "meta":  { ".read": true, ".write": true }
  }
}
🔒 เหมาะกับโปรโมชั่นภายในระยะสั้น เพราะมีเฉพาะคนที่รู้ลิงก์ + PIN เท่านั้นที่เข้าใช้ ถ้าต้องการให้ล็อกอัตโนมัติหลัง 30 ก.ย. 2026 บอกทีมงานได้ เดี๋ยวปรับกฎให้
